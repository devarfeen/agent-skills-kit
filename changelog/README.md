# Changelog Files

Store generated release notes and summaries in this folder.

## Naming
- Date-based release notes: `DDMMYYYY.md` (example: `12032026.md`)
- Session summary without explicit feature name: `DDMMYYYY.md`
- Feature-based summary: `<feature-name>.md` in lowercase kebab-case

## Required Structure
1. Date line (date-based mode only)
2. Project heading
3. Feature heading
4. Child sections: Summary, Problem, Change, Impact, Scope, Commits Included
