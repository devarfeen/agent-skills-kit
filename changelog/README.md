# Changelog Files

Store generated release notes and summaries in this folder.

## Naming
- Date-based release notes: `DD-Month-YYYY.md` (example: `12-March-2026.md`)
- Session summary without explicit feature name: `DD-Month-YYYY.md`
- Feature-based summary: `Feature-Name.md` using Title-Case words joined by hyphens

## Required Structure
1. Date line (date-based mode only)
2. Project heading
3. Feature heading
4. Child sections: Summary, Problem, Change, Impact, Scope, Commits Included
